Developing Amazon Recommendations

Harrison Li, Thomas Jiang, Masahiro Kusunoki, Daniel Chen

CS 109a Fall 2016

The Amazon Fine Foods Dataset, collected and distributed by the Stanford Network Analysis Project, contains nearly 600,000 Amazon reviews of over 70,000 food and food-related products by over 250,000 users. With this data, we set out to develop a system for predicting future ratings based on a combination of product features and user characteristics. If successful, such a system could be used to develop an accurate recommendation system.

Data Sources:

Our data set consists of 568,454 reviews on 74,258 food-related Amazon products by 217468 users. Each review includes UserId, ProductId, Score, and Text, as well as some other values (to see full list, with explanation, click the right arrow).

In order to obtain meaningful features for the 74,258 products in the data set, we took advantage of the Amazon API. This API allowed us to look up items by the ProductIds and return Product Groups. For example, product _[insert id]_ returns product group _[insert]_.

Data Exploration

We conducted several initial explorations of the dataset in order to develop a better understanding of trends in the data.

How People Review Products

First, we explored distributions of ratings. This allowed us to learn how sparse the data set was. A data set where every user reviewed a majority of the products and each product would have a large number of reviews would be ideal for modeling, but this will turn out not to be the case. Instead, this is an informative method for understanding what data will have to be filtered out and what data can be used directly from the data set.

Score Distribution

Interestingly, a large majority (63.9%) of the ratings given by users across all products are 5 out of 5, while negative ratings (defined as 3 stars or less) constitute only 22% of all ratings. The heavy skew towards 5 star ratings might present a challenge in distinguishing between products or users.

User Activity

Breaking down the users by number of reviews, it's clear that the vast majority of users review very few products. In fact, out of the 256059 unique users, over half of them only have one review! Since it is impossible to create a robust model with only a single data point for each user, we will need to filter for "active users", which we will define as users with greater than _____ reviews.

Products by Number of Reviews

Once again, we see that the majority of products have only been reviewed a small number of times. Out of the 74258 unique products, half have been reviewed two or fewer times. This indicates that we cannot simply use each product's score to assign a quality to it. Instead, we will either have to filter out products with too few reviews or use a prior to properly score products based on user ratings.
